2017 年前端培训计划

第一周 （二级标题）

主要是帮忙把 Web 开发的环境搞定，主要是编辑器、操作系统、命令行CLI、Git&Github 还有 Hexo 博客

星期六安排一个作业，就是做一个小博客的前端实现，比如现在你们看到的这个。

第二周

深入 JavaScript 的学习，从内置对象、语言核心、编程范式、异步流程、浏览器通信等方面切入，周末布置一个实战：一块电子钟或石英钟的 Canvas 实现。

内容包括以下：（列表）

  1. 内置对象，主要是 String Number Array Object Date 这些极为常用的内置对象
  2. 语言核心，从基本的关键字讲起，理解函数，理解作用域，进而理解作用域链，然后水到渠成的理解闭包
  3. 编程范式，虽然面向对象已经成为工业界普遍采用的编程范式，但是面向对象本身的缺陷是复杂性，状态关系模型会随着业务的增长变得难以预测，而函数式编程的不变性，高抽象性却不易造成复杂性的爆炸，其唯一的缺点是，起点较高，需要一定的数学基础。
  4. 异步流程，如果编程都是同步的，那该多无趣啊。
  5. 浏览器通信，与后台交换数据。

第三周

着重讲解前端工程化问题，打包到部署上线全一条龙

第四周

团队完成 Daily CSS 的开发

2017 年后台培训计划

第一周环境

环境问题，配好后台开发的工具，比如编辑器、OS、CLI、Git&Github等等，学习数据库并完成小博客的后台实现

第二周深入

跟前端一起上课

其次要加课：学习数据库、学习HTTP、学习一些网络知识、学习Linux运维

第三周集训

内容包括但不限于：

  • 数据库的使用
  • 数组及其函数式描述
  • 讨论树
    • S-表达式及其树的描述
    • 解释器处理S-表达式的全过程
    • 求 Tokens
    • 求 AST 抽象语法树
    • 对S-表达式求值
  • 对象是一类哈希表而数组是由对象实现的
  • 常用 Node 模块
  • 异步编程问题
  • 过程分析：
    • 构造过程抽象
    • 构造数据抽象
    • 套用范式
    • 模块化处理
  • 理解如下观点：
    • 函数是一等公民
    • 真实世界是一台趋于无穷的有限状态机
    • 数据的四个基本问题：增删查改
    • 递归和迭代的不同抽象层次
  • 复习整理一周所学

上述课程对所有人开放，前端也可以一起学习

团队完成 Daily CSS 的开发
